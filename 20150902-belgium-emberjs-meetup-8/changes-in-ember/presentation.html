What has changed in Ember world?

Releases

  • Ember.js: 1.13, 2.0
  • Ember Data: 1.0.0beta19, 1.13
  • ember-cli: 1.13

Lockstep development for Ember.js, Ember Data, Liquid Fire and ember-cli

  • Will follow the same versioning from 2.0
  • A new version of Ember.js will always be tested with the recent versions of the other libraries

Changes in Ember.js

  • Deprecations in 1.13
  • Errors in 2.0

Immutable data structures

  • Glimmer only re-renders the diff
  • Can now work with immutable data structures (e.g. sort returns a new array)
    • 
{{#each |items| as item}}
  {{timeline-item item=item}}
{{/each}}

Move towards components

  • Components are taking a prominent place
  • Long-term: move everything into components (routable components)
  • "Data in, events out" paradigm

No more views!

  • Views are deprecated in 1.13
  • Annoying cause routable components are not there yet
  • Wasn't clear what to replace a view with

IE8 no longer supported

Changes in Ember Data

Out of beta!

  • Last beta release was beta19
  • Jumped straight to 1.13
  • API changes but backwards compatible

JSON API v1.0

  • JSON API has more or less stabilized with the 1.0 release
  • Ember Data includes an adapter and serializer for JSON API

New behavior for find methods

  • First time store.find is called, fetch new data
  • Next time return cached data
  • Fetch new data in the background and update

Associations are async by default

HSTRY is moving to Chile
