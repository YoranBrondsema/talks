Testing in Ember.js

Testing is a first-class citizen in Ember.js

https://guides.emberjs.com/v2.3.0/testing/

With ember-cli, two ways:

  • in browser (development): http://localhost:4200/tests
  • in terminal (CI): ember test

In general, two levels of testing your application:

  • integration testing (slow): simulate real user, happy path
  • unit testing (fast): isolated functions, all paths (including errors)

Integration tests

  • Simulate a real user (visit page, click on link, click on button, test presence of HTML elements in DOM)
  • Perfect for simulating happy path
  • Implementation independent
  • Tests the integration between back-end and front-end
  • Slow

Unit tests

  • Test isolated functions
  • Allows to test scenarios that are hard/impossible to simulate (connection drop-off, server request fails)
  • Tightly coupled with the implementation
  • Fast

These concepts hold for an Ember.js application, with addition of Ember.js component tests

  • Isolated integration tests: no interaction with back-end so fast, but tests the behavior
