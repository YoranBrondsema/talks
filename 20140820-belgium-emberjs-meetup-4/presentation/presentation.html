What is Ember Data?

The "M" in MVC

Framework for managing your models and relationships

Separate project

  • Different repository
  • Can use Ember.js without Ember Data

Four main components

Models

Store

Adapter

Serializer

Specifying a model


App.Post = DS.Model.extend({
  /* Attributes */
  title: DS.attr('string'),
  body: DS.attr('string'),
  nrViews: DS.attr('number'),
  createdAt: DS.attr('date'),
  isPublished: DS.attr('boolean'),
  /* Relationships */
  author: DS.belongsTo('user'),
  comments: DS.hasMany('comment'),
  /* Properties */
  hasComments: function() {
    return this.get('comments.length') > 0;
  }.property('comments.length')
});

The store

  • Contains all instances of models
  • Is a singleton

Create and destroy instances

  • store.createRecord
  • store.deleteRecord

Query records

  • store.find('post', 1)
  • store.find('post')
  • store.find('post', { isPublished: true })
